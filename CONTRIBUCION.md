# Guía de contribución


## Subir modificaciones

Cada vez que hagas cambios, correr estos 3 comandos:

```bash
git add .
git commit -m "tipo: descripción corta"
git push
```

## Reglas de commits

| Tipo | Cuándo usarlo |
|------|---------------|
| `feat` | Agregás funcionalidad nueva |
| `fix` | Corregís un error |
| `docs` | Cambiás documentación (README, comentarios) |
| `refactor` | Reorganizás código sin cambiar lo que hace |
| `test` | Agregás o modificás tests |
| `chore` | Tareas de mantenimiento (gitignore, estructura) |

## Ejemplos de commits
feat: agregar módulo de pedidos

fix: corregir validación de teléfono en clientes

docs: actualizar README con instrucciones de uso

refactor: simplificar función buscar_por_id

test: agregar tests para módulo cobros

chore: actualizar .gitignore
